HTML Accessibility

Creating accessible web content for all users, including those with disabilities.

Introduction to Web Accessibility

Web accessibility means that websites, tools, and technologies are designed and developed so that people with disabilities can use them. More specifically, people can:

  • Perceive, understand, navigate, and interact with the Web
  • Contribute to the Web

Accessibility encompasses all disabilities that affect access to the Web, including:

  • Visual: blindness, low vision, color-blindness
  • Auditory: deafness and hard-of-hearing
  • Motor: inability to use a mouse, slow response time, limited fine motor control
  • Cognitive: learning disabilities, distractibility, inability to remember or focus on large amounts of information

Semantic HTML for Accessibility

Using semantic HTML elements is the foundation of web accessibility. They provide meaning to both browsers and assistive technologies.

Non-Semantic vs. Semantic HTML

Non-Semantic (Avoid)

<div class="header">
    <div class="title">Page Title</div>
</div>
<div class="nav">
    <div class="nav-item">Home</div>
    <div class="nav-item">About</div>
</div>
<div class="main">
    <div class="content">Main content here...</div>
</div>

Semantic (Recommended)

<header>
    <h1>Page Title</h1>
</header>
<nav>
    <ul>
        <li><a href="#">Home</a></li>
        <li><a href="#">About</a></li>
    </ul>
</nav>
<main>
    <article>Main content here...</article>
</main>

Key semantic elements include:

  • <header>, <footer>, <nav>, <main>, <article>, <section>, <aside>
  • Proper heading hierarchy (<h1> through <h6>)
  • Lists (<ul>, <ol>, <li>) for grouped items
  • <button> for interactive elements (not styled <div>s)

ARIA (Accessible Rich Internet Applications)

ARIA is a set of attributes that define ways to make web content and applications more accessible to people with disabilities.

Key ARIA Concepts

  • Roles: Define what an element is or does
  • Properties: Define characteristics of elements
  • States: Define current conditions of elements

ARIA Examples

1. Landmark Roles

<div role="navigation" aria-label="Main Navigation">
    <!-- Navigation content -->
</div>

<div role="search">
    <!-- Search form -->
</div>

2. Live Regions

<div aria-live="polite" aria-atomic="true">
    <!-- Content that will update dynamically -->
</div>

3. Form Accessibility

<input type="text" aria-required="true" aria-invalid="false" aria-describedby="name-hint">
<div id="name-hint">Please enter your full name</div>

ARIA Best Practices

  1. Use native HTML elements with built-in accessibility features when possible
  2. Don't change the semantics of HTML elements unless absolutely necessary
  3. All interactive elements should be keyboard accessible
  4. All interactive elements should have an accessible name
  5. Don't use role="presentation" or aria-hidden="true" on elements that are focusable

Accessible Forms

Forms are often challenging for users with disabilities. Here are key practices for creating accessible forms:

Accessible Form Example

Please enter a valid email address in the format: name@example.com
Preferred Contact Method:

Key Form Accessibility Features:

  • Proper <label> elements associated with form controls
  • Grouped related form elements with <fieldset> and <legend>
  • Clear instructions and error messages
  • ARIA attributes for additional context
  • Logical tab order

Keyboard Navigation

Many users with disabilities rely on keyboards or keyboard alternatives to navigate websites. Ensure all functionality is available using only a keyboard.

Key Keyboard Accessibility Features:

  • Focus indicators: Visible focus styles for all interactive elements
  • Logical tab order: Elements receive focus in an order that follows the content's meaning
  • Skip links: Allow users to bypass repetitive navigation
  • Keyboard traps: Avoid situations where keyboard users can't move focus away from an element

Color and Contrast

Proper color contrast is essential for users with low vision or color blindness.

This text has poor contrast and is difficult to read.

This text has good contrast and is easier to read.

This text has high contrast for maximum readability.

Color Accessibility Guidelines:

  • Maintain a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text
  • Don't rely on color alone to convey information
  • Provide additional indicators (icons, patterns, text) alongside color
  • Consider how your design appears to users with color blindness
