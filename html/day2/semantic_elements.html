Semantic HTML Elements

Semantic HTML elements clearly describe their meaning to both the browser and the developer. This page demonstrates the proper use of semantic HTML5 elements.

Non-Semantic vs. Semantic HTML

Non-Semantic HTML

Uses generic elements that don't describe their content:

<div class="header">
    <div class="nav">...</div>
</div>

<div class="main">
    <div class="article">...</div>
    <div class="sidebar">...</div>
</div>

<div class="footer">...</div>

Semantic HTML

Uses elements that clearly describe their purpose:

<header>
    <nav>...</nav>
</header>

<main>
    <article>...</article>
    <aside>...</aside>
</main>

<footer>...</footer>

Semantic HTML improves accessibility, SEO, and code readability.

Text-Level Semantic Elements

HTML5 provides several elements for marking up text with specific meaning:

  • <time> - Represents a specific period in time: June 15, 2023, 2:30 PM
  • <mark> - Represents text that is highlighted for reference: This text is marked
  • <cite> - Represents the title of a work: HTML5 Specification
  • <abbr> - Represents an abbreviation: W3C (hover to see the full title)
  • <dfn> - Represents a definition term: HTML is the standard markup language for documents designed to be displayed in a web browser.
  • <code> - Represents computer code: const x = 10;
  • <samp> - Represents sample output: Error: File not found
  • <kbd> - Represents keyboard input: Press Ctrl + C to copy
  • <var> - Represents a variable: The equation is E = mc2

Accessibility Benefits

Using semantic HTML elements provides several accessibility benefits:

  • Screen readers can better understand the page structure and content
  • Keyboard navigation is improved with proper semantic elements
  • Search engines can better understand and index your content
  • Developers can more easily understand and maintain the code
  • Assistive technologies can provide better user experiences

For example, a screen reader can announce a navigation area when it encounters a <nav> element, or it can allow users to skip to the main content when it finds a <main> element.

Best Practices

  • Choose semantic elements based on their meaning, not their appearance
  • Use <section> for thematic grouping of content
  • Use <article> for self-contained, independently distributable content
  • Use <nav> for major navigation blocks
  • Use <header> and <footer> for introductory and concluding content
  • Use <aside> for content that is tangentially related to the main content
  • Use <figure> and <figcaption> for self-contained content like images with captions
  • Use <main> to denote the main content area (only one per page)
  • Avoid using <div> when a more semantic element is appropriate
